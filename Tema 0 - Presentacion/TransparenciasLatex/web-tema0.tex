\section{Objetivos y Temario}

\subsection{Objetivos}

\begin{frame}[c]
   \frametitle{Objetivos de la Asignatura}
   \begin{enumerate}[<+->]
        \item Comprender el concepto de aplicación empresarial.
        \item Comprender cómo se divide una aplicación empresarial en capas.
        \item Saber aplicar patrones de diseño para el desarrollo de la capa de negocio de una aplicación empresarial.
        \item Conocer, comprender y saber configurar patrones de diseño para la creación de puentes objeto-relacional.
        \item Saber utilizar herramientas para la generación automática de puentes objeto-relacional.
        \item Saber aplicar patrones de diseño para el desarrollo de la capa de servicio de una aplicación empresarial.
        \item Saber especificar y diseñar una capa de servicios REST.
        \item Saber aplicar patrones de diseño para el desarrollo de la capa de presentación de una aplicación empresarial.
        \item Ser capaz de utilizar algunos de los servicios proporcionados por los servidores de aplicaciones, tales como seguridad.
	\end{enumerate}
\end{frame}

\begin{frame}[c]
   \frametitle{Resultados de Aprendizaje}
   El alumno será capaz de:
   \begin{enumerate}[<+->]
        \item Elaborar modelos de dominio utilizando POJOs.
        \item Anotar un modelo de dominio para especificar cómo debe ser persistido utilizando JPA.
        \item Utilizar las facilidades de Spring para la construcción de capas de persistencia.
        \item Definir una interfaz REST que permita la manipulación de un determinado modelo de dominio.
        \item Ser capaz de serializar objetos de dominio en formato JSON usando anotaciones Java.
        \item Ser capaz de implementar una controlador HTTP usando Spring.
        \item Ser capaz de implementar una capa de servicio usando POJOS.
        \item Ser capaz de implementar una interfaz SPA utilizando AngularJS.
        \item Ser capaz de utilizar tokens de autenticación.
        \item Ser capaz de utilizar los servicios proporcionados por Spring para el control de accesos.
	\end{enumerate}
\end{frame}

	\frametitle{Cálculo de la Calificación Final}
	\begin{block}{Fórmula de Cálculo de la Calificación Final}
		\begin{align*}
			Calificacion Final  =  & Proyecto \times 0.75 + \\
                                   & Prueba Final Escrita \times 0.25 \\
		\end{align*}
	\end{block}
	\begin{itemize}
		\item<2-> Para poder aplicar la fórmula anterior se debe obtener un mínimo de 5.00 en el proyecto.
        \item<3-> En virtud del transcurso de la asignatura, se podrá eximir a determinados alumnos de la realización de la prueba final escrita.
	\end{itemize}
\end{frame}

\subsection{Prueba Final}

\begin{frame}[c]
    \frametitle{Prueba Final}
	\begin{enumerate}[<+->]
		\item Prueba evaluable escrita en las fechas asignadas por el centro.
		\item La prueba contendrá ejercicios y cuestiones sobre razonamientos teóricos.
        \item Se podrá hacer uso todo tipo de material escrito.
        \item En ningún caso se dejará hacer uso de de dispositivos electrónicos, especialmente, de aquellos con capacidades de comunicación inalámbrica.
		\item \alert{El material escrito debe servir para consultar cuestiones puntuales, pero en el caso ideal no debería hacerse ningún uso de los mismos}.
		\item Aquellas respuestas que sean una simple copia de lo disponible en algún tipo de material de la asignatura se calificarán con 0 puntos.
	\end{enumerate}
\end{frame}
